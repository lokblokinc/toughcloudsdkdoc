What is Toughcloud

Toughcloud is a service offering from LokBlok that offers secure multicloud storage for data, backup and recovery.

Toughcloud is a sharding storage platform where data is split and stored on multiple cloud platforms creating a resilient and highly secure data storage, backup and recovery system. Data �blobs� are broken up and stored using a k of n algorithm similar to the way RAID storage uses Reed Solomon and other algorithms to securely store data across multiple storage systems. Stored digital assets cannot be accessed from a single cloud service but requires the recovery of digital asset pieces from multiple clouds to reconstitute the original digital asset using the user�s private key (that is also reconstituted within a Toughkey hardware security module).

Toughcloud typically works in conjunction with Toughbox, which is a Key recovery without custody platform service offering from LokBlok. Toughbox provides developers with an easily implementable system to allow secure, non-custodial backup and recovery by assigning trusted �k of n� recovery agents.

Overview

The Toughcloud SDK is a set of .NET libraries with the classes and interfaces allowing secure storage. It also contains fully working and integrated with Toughbox sample code demonstrating usage from JavaScript.

To call/invoke these from Java script one method is to use node-api-dotnet npm package which enables interoperability between .NET and JavaScript in the same process.

See GitHub - microsoft/node-api-dotnet: Advanced interoperability between .NET and JavaScript in the same process.

Sample Application

A sample project and steps to create it is provided for reference. It demonstrates the steps to:

  1. Setup a Toughcloud user account with Tougbox key recovery enabled.

  2. Test Toughcloud file encrypt-upload, download-decrypt, file status and file delete
    functionality.

  3. Each file/user has an associated set of encryption and signature keys which are enrolled with Toughbox for recovery.

  4. Simulate Toughcloud recovery by first encrypt-uploading a file, then deleting the
    local user keys, followed by a key recovery and subsequent download-decrypt of the
    file uloaded prior to recovery.

Before building and running the sample application be sure to configure the correct user
and credential information for the string variables used throughout the sample.
