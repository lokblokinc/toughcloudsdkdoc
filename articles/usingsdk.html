Using Toughcloud with Toughbox SDK

This SDK combines Toughcloud with Toughbox creating a file storage solution that includes key recovery for the user's Toughcloud keys.

Before a user can access Toughcloud they must first have a Toughbox account. The Toughbox user's account must be fully setup with their Recovery Agents. The Toughcloud user initialization then follows/

The typical flow for Toughbox/Toughcloud for this SDK is as follows:

  1. Setup Toughbox user account with Recovery Agents (prerequisite). Refer to the Toughbox documentation for help with this.

  2. Create Toughcloud User on local machine using this SDK.

  3. Use response from step 2 to Activate Toughcloud Secret in Toughbox.
    This effectively enrolls the Toughcloud User for Toughcloud services via Toughbox Management API (not in this document).

  4. Prior to any Toughcloud FileUpload or FileDownload operation, obtain an Authorization Token from Toughbox Management API (not in this document)

  5. Upload and Download files on local machine using this SDK along with the Authorization Token obtained from Tougbox.

  6. If Toughcloud User keys are lost or the key password forgotten, keys may be recovered using the Toughcloud..Recover.. APIs in this SDK.

Create User

Before a file can be uploaded and downloaded in Toughcloud, the user's Toughcloud account must be created on their local machine. The data resulting from the create method is used to enroll the user for Toughcloud services as well as enrolling their Toughcloud keys with their Toughbox account for recovery. The secret name that is specified as part of initializing the user account must be a fixed string i.e., the SDK caller should use the same string for the Toughcloud application throughout. Any given Toughcloud user will only ever need a single secret for the Toughcloud application.

Upload and Download Files

Prior to calling the API to upload or download files, an authentication token must be generated from the Toughbox service (details are in the Toughbox API documentation for the AuthorizeToughcloudAction REST API). This authorization token must be include in the API to upload or download files.
